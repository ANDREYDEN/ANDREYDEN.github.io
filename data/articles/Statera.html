Background

Statera is a an app that helps you distribute expenses within a group.

Originally, this project started as a solution to a problem my friends and I had while living together: how do you keep track of shared expenses when there are different people buying and consuming different things? At first we used a spreadsheet to keep track of this, but it soon became too cumbersome and error prone, so the idea of creating an app to solve this problem emerged naturally.

The app is heavily inspired by my friends' capstone project and a similar app in this field called Splitwise.

The frontend of the app is written in Flutter which allowed for multiplatform support out of the box: it's available on Android (Google Play Store), iOS (iOS App Store), MacOS (MacOS App Store), WEB (the website) with planned support for Windows and Linux down the line.

Firebase was used as a backend as a service solution to store data, handle authentication and host the website. The app functionality includes:

  • notifications (Firebase Cloud Messaging)
  • in-app update prompts (Firebase App Distribution)
  • welcome dialogs (Firebase Remote Config)
  • deep linking (Firebase Dynamic Links)

A key feature of Statera is the ability to import receipt data from a photo. The app uses Google Cloud Vision API to extract text from images and then parses the text to extract item name, price and other useful metadata. Currently, all of the OCR heavy lifting is happening in a Firebase Cloud Function while the app also uses Firebase Storage to store the images.

What's inside?

Flutter, Dart, hosted on Firebase, OCR implemented with Google Cloud Vision API
